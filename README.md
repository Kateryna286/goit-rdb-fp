### Імпорт даних

Імпорт даних було виконано за допомогою Import Wizard у MySQL Workbench відповідно до інструкцій курсу. Після імпорту таблиця `infectious_cases` містить не всі записи з CSV-файлу, оскільки частина рядків не була завантажена через наявність порожніх або некоректних значень у деяких колонках. Import Wizard у MySQL Workbench пропускає такі рядки без зупинки процесу імпорту.

Подальша нормалізація та аналіз виконуються на фактично імпортованому наборі даних. Кількість завантажених записів зафіксована за допомогою запиту `SELECT COUNT(*)`.

У межах першого етапу проєкту було виконано імпорт даних у схему pandemic. Для усунення дублювання атрибутів Entity та Code створено окрему таблицю entities з унікальними значеннями. На основі вихідної таблиці infectious_cases побудовано таблицю infectious_cases_no_entity_code, у якій замість текстових атрибутів використовується ідентифікатор entity_id.

У межах другого етапу проєкту дані було нормалізовано до третьої нормальної форми (3НФ). Для цього:
- створено довідникову таблицю `diseases` з унікальними назвами захворювань;
- побудовано таблицю `infectious_cases_normalized`, у якій кожен запис відповідає комбінації країна–рік–захворювання.

На етапі нормалізації структура даних змінювалася без очищення або фільтрації значень. Порожні значення кількості захворювань збережено у вигляді порожніх рядків, оскільки їх відсіювання виконується на етапі аналітичних запитів відповідно до умов наступних завдань.
